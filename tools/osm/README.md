# OSM Data Acquisition & Conversion — Story 6.1

Dev-time tooling only. Nothing under `tools/` ships in the APK — the *outputs* of these
scripts are what get bundled (a reseeded `campus.db` and `ui/src/main/assets/tiles/`).
This directory exists so the acquisition/conversion process is reproducible and
reviewable, per epics.md's own framing of Story 6.1 ("pull the extract once, run two
separate conversion scripts against it") — a deliberate, documented exception to this
project's usual "no committed script files, just prose-documented `sqlite3` CLI commands"
convention (Stories 1.1-5.2), justified here by data volume: 178 real buildings and 40
real walkway ways is not something that can be hand-typed as `sqlite3` INSERT statements
the way 5-10 seed rows were in prior stories.

## 1. Data source

OpenStreetMap, via the Overpass API. Endpoint used: `https://overpass.openstreetmap.fr/api/interpreter`
(the flagship `overpass-api.de` endpoint returned a persistent `Dispatcher_Client` runtime
error for every query tried, including trivial known-good ones, during this session —
not specific to our query; the French mirror, part of the same federated Overpass
infrastructure, same database, was used instead and worked reliably).

**Bounding box** — not a hand-picked guess. Nominatim resolved "University of the
Witwatersrand" to `amenity=university` way `26327830`, a real, community-maintained
campus-outline polygon in OSM. Its bounding box is the bounding box used for every
extraction query below:

```
south=-26.1945  west=28.0195  north=-26.1835  east=28.0338
```

(the way's own exact bbox, `-26.1939535,-26.1840236` to `28.0200128,28.0332544`, padded
by a small buffer to catch any edge-straddling geometry).

## 2. Extraction queries (raw/ — committed for reproducibility)

Run 2026-07-13. Three separate Overpass QL queries, one per data need:

**`raw/wits_buildings.json`** — every building footprint in the bbox (178 ways):
```
[out:json][timeout:80];
way(-26.1945,28.0195,-26.1835,28.0338)[building];
out tags geom;
```

**`raw/wits_walkways_topo.json`** — the walkway graph, with real OSM node topology
preserved (`(._;>;); out body;`, not `out geom`, specifically so shared nodes between
intersecting ways keep the same id — required to build a *connected* graph rather than
253 disconnected polylines):
```
[out:json][timeout:80];
way(-26.1945,28.0195,-26.1835,28.0338)[highway~"^(footway|path|steps)$"];
(._;>;);
out body;
```
40 ways, 253 unique nodes.

**`raw/wits_terrain.json`** — roads and land-cover context for Story 6.2's basemap tiles
(576 ways: `highway=*`, `landuse=*`, `natural=*`, `leisure=*`):
```
[out:json][timeout:80];
(way(-26.1945,28.0195,-26.1835,28.0338)[highway];
 way(-26.1945,28.0195,-26.1835,28.0338)[landuse];
 way(-26.1945,28.0195,-26.1835,28.0338)[natural];
 way(-26.1945,28.0195,-26.1835,28.0338)[leisure];);
out tags geom;
```

Data license: ODbL (OpenStreetMap Foundation) — attribution requirement is carried
through to Story 6.2's map attribution AC.

## 3. Conversion scripts

- `convert_walkways.py` — `raw/wits_walkways_topo.json` → `output/node_edge_reseed.sql`
  (Node/Edge `DELETE`+`INSERT` statements) and `output/building_coords_update.sql`
  (3 `Building.latitude/longitude` corrections — see script docstring for why).
- `convert_footprints.py` — `raw/wits_buildings.json` → `output/building_footprints.geojson`
  (per-Building polygon set, matched to the 5 existing seed Buildings — 3 of 5 matched,
  see script docstring). Ready for Story 6.3 to seed into its new `BUILDING_FOOTPRINT`
  entity; this story does not touch that entity (doesn't exist yet).
- `generate_tiles.py` — `raw/wits_terrain.json` → `../../ui/src/main/assets/tiles/{z}/{x}/{y}.pbf`
  (real Mapbox Vector Tile protobuf tiles, zoom 14-18, generated with `mapbox-vector-tile`
  + `mercantile` + `shapely` — no `tippecanoe`/`gdal` available in this sandboxed
  environment; see script docstring for the tile-encoding approach). This is the
  "bundled offline vector-tile source, committed as a build-time asset" AC1 asks for;
  Story 6.2 wires it into `style.json` as a *declared* source.

Run order: `python3 convert_walkways.py && python3 convert_footprints.py && python3 generate_tiles.py`.
All three are independent of each other's output (all read only from `raw/`).

## 4. Applying to `campus.db`

```
sqlite3 ../../data/src/main/assets/database/campus.db < output/node_edge_reseed.sql
sqlite3 ../../data/src/main/assets/database/campus.db < output/building_coords_update.sql
```

Same `sqlite3`-CLI-authoring convention every prior story used (Stories 1.1-5.2) — the
difference here is the SQL is generated by a script instead of hand-typed, because of
the row count, not a change to the underlying convention.
